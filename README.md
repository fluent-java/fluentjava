# FluentJava

FluentJava is a compile-time `javac` plugin and runtime utility library that adds fluent-style methods to core Java types while keeping production bytecode explicit, safe, and standard.

- Compile-time rewrite only (`-Xplugin:FluentJava`)
- Runtime uses pure static methods
- No runtime agent, no bytecode manipulation at runtime
- Java 17+

---

## Five Practical Examples

### 1) String validation and normalization
```java
String username = rawUsername.trimToNull();
if (username == null || !username.isAlphanumeric()) {
    throw new IllegalArgumentException("Invalid username");
}
```

### 2) Safe parsing pipeline
```java
Integer age = rawAge.trimToNull().toIntOrNull();
if (age == null || !FluentNumber.isBetween(age, 18, 120)) {
    throw new IllegalArgumentException("Invalid age");
}
```

### 3) Readable list transformations
```java
List<String> premiumEmails = users
        .filterBy(User::isPremium)
        .mapTo(User::getEmail)
        .filterNotNull()
        .distinctBy(String::toLowerCase)
        .sortedBy(String::toLowerCase);
```

### 4) Reporting by groups
```java
Map<String, List<Order>> byStatus = orders.groupBy(Order::getStatus);
double total = orders.sumOf(Order::getAmount);
double average = orders.averageOf(Order::getAmount);
```

### 5) Date rules in business logic
```java
LocalDate invoiceDate = LocalDate.now().minusDays(10);
boolean overdue = FluentDate.daysUntil(invoiceDate, LocalDate.now()) > 7;
boolean payableToday = FluentDate.isWeekday(LocalDate.now()) && !FluentDate.isWeekend(LocalDate.now());
```

---

## How It Works

You write fluent calls such as `names.firstOrNull()` or `"42".toIntOrNull()`.
At compile-time, FluentJava rewrites them to explicit static calls like `FluentList.firstOrNull(names)` and `FluentString.toIntOrNull("42")`.

Generated bytecode remains plain Java `invokestatic` instructions.

## Quick Start

### Build
```bash
cd fluentjava
mvn clean install
```

### Maven setup
```xml
<dependencies>
    <dependency>
        <groupId>io.fluentjava</groupId>
        <artifactId>fluentjava-runtime</artifactId>
        <version>1.0.0</version>
    </dependency>
    <dependency>
        <groupId>io.fluentjava</groupId>
        <artifactId>fluentjava-plugin</artifactId>
        <version>1.0.0</version>
        <scope>provided</scope>
    </dependency>
</dependencies>

<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.12.1</version>
            <configuration>
                <compilerArgs>
                    <arg>-Xplugin:FluentJava</arg>
                    <arg>-J--add-exports=jdk.compiler/com.sun.tools.javac.api=ALL-UNNAMED</arg>
                    <arg>-J--add-exports=jdk.compiler/com.sun.tools.javac.tree=ALL-UNNAMED</arg>
                    <arg>-J--add-exports=jdk.compiler/com.sun.tools.javac.util=ALL-UNNAMED</arg>
                    <arg>-J--add-exports=jdk.compiler/com.sun.tools.javac.code=ALL-UNNAMED</arg>
                </compilerArgs>
                <fork>true</fork>
            </configuration>
        </plugin>
    </plugins>
</build>
```

### FluentValidator
- `of`: Create a validator for any value type.
- `ofString`: Create a validator specialized for strings.
- `notNull`: Add a non-null rule.
- `satisfies`: Add a custom predicate rule with message.
- `notBlank`: Add a string blank check rule.
- `minLength`: Add a minimum string length rule.
- `maxLength`: Add a maximum string length rule.
- `matches`: Add a regex rule for strings.
- `notEmpty`: Add a non-empty rule for String, Collection, or Map.
- `min`: Add a numeric minimum rule.
- `max`: Add a numeric maximum rule.
- `positive`: Add a strictly positive numeric rule.
- `validate`: Execute all rules and return all error messages.
- `isValid`: Return true when there are no errors.
- `throwIfInvalid`: Throw `IllegalArgumentException` with aggregated messages.

`FluentValidator` uses a Builder pattern with internal rules as `List<Supplier<Optional<String>>>`.
It is used directly (`FluentValidator.of(...)`) and is intentionally not part of static-import fluent method rewriting.

## ★ Extending FluentJava — Step by Step

The architecture is designed so adding a new method requires **minimum file changes**.

### Adding a new method on an EXISTING type (e.g., `reverse()` on String)

**Only 2 files to edit:**

#### Step 1 — Runtime: Add the static method

In `fluentjava-runtime/.../FluentString.java`:

```java
/**
 * Reverses the string. Returns null if input is null.
 */
public static String reverse(String s) {
    if (s == null) return null;
    return new StringBuilder(s).reverse().toString();
}
```

#### Step 2 — IntelliJ plugin: Add ONE registry entry

In `FluentMethodRegistry.java`, add to the `ALL_METHODS` list:

```java
method("reverse", TargetType.STRING)
    .returns("java.lang.String")
    .doc("Reverses the string. Returns <code>null</code> if null.",
         "\"hello\".reverse() → \"olleh\"")
    .build(),
```

**That's it!** Everything else is automatic:
- ✅ **Javac plugin**: Auto-discovers `reverse` via reflection — zero changes
- ✅ **IntelliJ PSI**: Auto-injects `reverse()` as a virtual method on String
- ✅ **Autocompletion**: Shows `reverse()` after typing `.` on a String
- ✅ **Go To Declaration**: Ctrl+B navigates to `FluentString.reverse()`
- ✅ **Documentation**: Ctrl+Q shows the description and example
- ✅ **No red errors**: The PsiAugmentProvider handles it

#### Step 3 — Build & test

```bash
# Rebuild runtime + javac plugin
cd fluentjava && mvn clean install

# Rebuild IntelliJ plugin
cd fluentjava-intellij-plugin && gradlew clean build
```

### Adding a new method WITH parameters

```java
// Runtime:
public static String padLeft(String s, int length, char padChar) {
    if (s == null) return null;
    if (s.length() >= length) return s;
    return String.valueOf(padChar).repeat(length - s.length()) + s;
}

// Registry:
method("padLeft", TargetType.STRING)
    .param("length", "int")
    .param("padChar", "char")
    .returns("java.lang.String")
    .doc("Pads the string on the left to reach the given length.",
         "\"42\".padLeft(5, '0') → \"00042\"")
    .build(),
```

### Adding a new generic method on List

```java
// Runtime:
public static <T> List<T> distinctBy(List<T> list) {
    if (list == null) return Collections.emptyList();
    return list.stream().distinct().toList();
}

// Registry:
method("distinctBy", TargetType.LIST)
    .returnsTargetType()   // Returns List<E>
    .doc("Removes duplicates from the list.",
         "List.of(1,2,2,3).distinctBy() → [1,2,3]")
    .build(),
```

### Adding a method with its own type parameter (like mapTo)

```java
// Runtime:
public static <T, R> List<R> flatMapTo(List<T> list, Function<T, List<R>> fn) {
    if (list == null) return Collections.emptyList();
    return list.stream().flatMap(t -> fn.apply(t).stream()).toList();
}

// Registry:
method("flatMapTo", TargetType.LIST)
    .typeParam("R")   // Declares <R> on the method
    .param("mapper", "java.util.function.Function", "E", "java.util.List<R>")
    .returns("java.util.List", "R")   // Returns List<R>
    .doc("Flat-maps each element.",
         "groups.flatMapTo(Group::getMembers)")
    .build(),
```

### Adding a completely new target type (e.g., FluentMap)

**4 steps:**

1. **Create** `FluentMap.java` in `fluentjava-runtime`:

```java
package io.fluentjava.map;

public final class FluentMap {
    public static <K,V> V getOrDefault(Map<K,V> map, K key, V defaultValue) {
        return (map == null || !map.containsKey(key)) ? defaultValue : map.get(key);
    }
}
```

2. **Register** in `META-INF/fluentjava/targets`:

```
io.fluentjava.string.FluentString
io.fluentjava.list.FluentList
io.fluentjava.map.FluentMap        ← add this line
```

3. **Add target type** in `FluentMethodRegistry.java`:

```java
public enum TargetType {
    STRING("java.lang.String", "io.fluentjava.string.FluentString"),
    LIST("java.util.List", "io.fluentjava.list.FluentList"),
    MAP("java.util.Map", "io.fluentjava.map.FluentMap"),  // ← add this
}
```

4. **Add method entries** in `ALL_METHODS`:

```java
method("getOrDefault", TargetType.MAP)
    .param("key", "K")
    .param("defaultValue", "V")
    .returnsElementType()
    .doc("Gets the value for the key, or the default if absent.",
         "config.getOrDefault(\"timeout\", 30)")
    .build(),
```

## Registry Builder API Reference

| Builder method | Effect | Example |
|----------------|--------|---------|
| `.returns("type")` | Sets return type (FQN) | `.returns("boolean")`, `.returns("java.lang.String")` |
| `.returns("type", "T1", "T2")` | Generic return type | `.returns("java.util.List", "R")` |
| `.returnsElementType()` | Returns E (collection element) | Used for `firstOrNull`, `lastOrNull` |
| `.returnsTargetType()` | Returns target parameterized with E | Used for `filterBy` (returns `List<E>`) |
| `.param("name", "type")` | Adds a parameter | `.param("count", "int")` |
| `.param("name", "type", "T1")` | Generic parameter | `.param("pred", "java.util.function.Predicate", "E")` |
| `.typeParam("R")` | Method-level type parameter | Used for `mapTo`'s `<R>` |
| `.doc("desc", "example")` | Documentation content | `.doc("Reverses the string.", "\"hi\".reverse()")` |

### Type tokens

| Token | Meaning | Resolved to |
|-------|---------|-------------|
| `"boolean"`, `"int"`, etc. | Primitive types | `PsiTypes.booleanType()`, etc. |
| `"java.lang.String"` | Class type | `PsiClassType` from factory |
| `"E"` | Element type of target collection | Target class's type parameter |
| `"R"`, `"K"`, `"V"`, etc. | Method's own type parameter | `LightTypeParameterBuilder` |

## How the Javac Plugin Auto-Discovery Works

The javac plugin uses **zero hardcoded method lists**. Instead:

1. `META-INF/fluentjava/targets` lists FluentXxx class FQNs
2. `FluentTargetRegistry` reads this file and uses **reflection** to discover all `public static` methods
3. `FluentMethodRewriter` rewrites only method names found in the registry
4. `FluentImportInjector` injects `import static` for each discovered class

**Result**: Add a new static method to `FluentString` → javac plugin automatically rewrites calls to it. Zero files to change in the plugin.

---

## How the IntelliJ Plugin Auto-Derivation Works

The IntelliJ plugin uses **one central registry** (`FluentMethodRegistry`):

1. `FluentMethodRegistry` defines all methods using a declarative builder API
2. `FluentJavaPsiAugmentProvider` iterates registry entries and builds `LightMethodBuilder` objects with proper types (generics, primitives, etc.) — **zero per-method code**
3. `FluentJavaCompletionContributor` iterates registry entries to create completion items — **zero hardcoded switches**
4. `FluentJavaGotoDeclarationHandler` uses the registry to resolve navigation targets
5. `FluentJavaDocumentationProvider` uses the registry for doc content

**Result**: Add one entry to `ALL_METHODS` → PSI injection, completion, goto, doc all work automatically.

## FAQ

**Q: How many files do I need to change to add a new method?**
A: **2 files**: The runtime class (add static method) + `FluentMethodRegistry.java` (add one builder entry). The javac plugin needs zero changes.

**Q: Is this like Lombok?**
A: Similar concept (compile-time), but different mechanism. FluentJava uses the official `com.sun.source.util.Plugin` API. The IntelliJ plugin uses the same `PsiAugmentProvider` technique that Lombok uses.

**Q: Do I need annotations?**
A: No. Zero annotations. Just `‑Xplugin:FluentJava` in your compiler config.

**Q: What if I decompile the bytecode?**
A: You'll see `FluentString.isBlankSafe("test")` — standard static calls. The fluent syntax exists only in source code.
